ส่วนที่ 1

คู่มือสำหรับผู้ใช้ทั่วไป

เนื้อหาในส่วนนี้สำหรับลูกค้าที่ต้องการเรียนรู้วิธีซื้อ Whitelist, ตรวจสอบสิทธิ์เข้าเกม, อ่านข่าวสาร และคำถามที่พบบ่อย — ใช้ได้ทุกอุปกรณ์ (มือถือ / แท็บเล็ต / คอมพิวเตอร์)

1.1 ภาพรวมและเมนูหลัก

แถบเมนูด้านบน (Navbar)

ติดอยู่ที่ด้านบนของทุกหน้า เลื่อนตามขึ้น/ลงไม่หาย ประกอบด้วย:

  • โลโก้ Judyshop (ซ้ายสุด) — คลิกเพื่อกลับหน้าแรก
  • เมนูหลัก: Home · Shop · News — เมนูที่กำลังเปิดอยู่จะมีจุดสีชมพูเล็กๆ ใต้ชื่อ
  • ปุ่มสลับภาษา TH / EN — ปุ่มสีชมพูคือภาษาที่เลือกอยู่ คลิกอีกปุ่มเพื่อสลับ (URL จะเปลี่ยนเป็น /en/... เมื่ออยู่ภาษาอังกฤษ)
  • ปุ่มเปลี่ยนธีม สว่าง/มืด
  • ไอคอน Discord และโซเชียลอื่น — เปิดในแท็บใหม่
บนมือถือ

เมนู Home/Shop/News จะซ่อนอยู่ในปุ่ม Hamburger (สามขีด) ทางขวา — แตะเพื่อเปิด Drawer แบบเลื่อนลงมา

Footer ด้านล่าง

เลื่อนสุดหน้าจะเจอ Footer ประกอบด้วย:

  • ลิขสิทธิ์ © ปีปัจจุบัน
  • ลิงก์ Check Whitelist
  • ไอคอนโซเชียล

1.2 หน้าแรก (Home)

หน้าแรกของเว็บไซต์ที่ / หรือ /en

สิ่งที่จะเห็น

  • โลโก้ JUDY SHOP ขนาดใหญ่ มี Effect เรืองแสง (สีชมพู + ฟ้า)
  • คำโปรย ใต้โลโก้
  • ปุ่ม Shop Now — ไปที่หน้า Shop
  • ปุ่ม Join Discord — เปิด Discord ในแท็บใหม่
  • Mascot น้อง Judy ลอยขยับเบาๆ ที่ด้านล่าง

ป็อปอัพประกาศ (ถ้ามี)

เมื่อแอดมินตั้งค่า Announcement ไว้ ป็อปอัพจะแสดงทันทีที่เปิดหน้าแรก:

  • มี 3 รูปแบบ: รูปภาพอย่างเดียว · ข้อความอย่างเดียว · ทั้งรูปและข้อความ
  • ปิดได้ 2 วิธี: กดปุ่ม X (มุมขวาบน) หรือคลิกพื้นที่มืดด้านหลัง
  • ป็อปอัพจะแสดงทุกครั้งที่เข้าหน้านี้ (ไม่จำว่าปิดไปแล้ว) — ทำให้ลูกค้าไม่พลาดประกาศสำคัญ

1.3 Shop — เลือกสินค้าและสั่งซื้อ

หน้ารวมสินค้าทั้งหมดที่ /shop

หน้า Shop หลัก

  • หัวเรื่อง SHOP พร้อมคำบรรยายใต้หัวเรื่อง
  • กริดสินค้า: มือถือ 2 คอลัมน์ · แท็บเล็ต 3 · คอมพิวเตอร์ 4

การ์ดสินค้าแต่ละใบแสดง

  • รูปสินค้า (สัดส่วน 5:3) — เคลื่อนซูมเล็กน้อยเมื่อ hover
  • ป้ายแบดจ์มุมขวาบน เช่น HOT NEW SALE (ถ้ามี)
  • ถ้ายังไม่เปิดขาย จะมีแถบทาบ Coming Soon ทับ และคลิกไม่ได้
  • ชื่อสินค้า + คำบรรยายสั้น (2 บรรทัด)
  • ราคาเริ่มต้น From ฿XXX / $XX (ราคาแผนถูกสุด)

เมื่อคลิกสินค้า — ป็อปอัพรายละเอียดสินค้า

  1. ดูรูปสินค้า — เลื่อนซ้าย/ขวาในแกลเลอรีเพื่อดูภาพหลายภาพ
  2. เลือกแพ็กเกจ — แท็บแต่ละอัน เช่น "30 วัน", "60 วัน", "ตลอดชีพ" — แต่ละแท็บแสดงราคาและระยะเวลา
  3. เลือกวิธีชำระเงิน — มี 2 ตัวเลือก:
    • PromptPay — สแกน QR ไม่มีค่าธรรมเนียม
    • บัตรเครดิต — มีค่าธรรมเนียมเพิ่ม (ถ้าตั้งค่าไว้) — แสดงในใบเสร็จย่อย
  4. กรอก Roblox Username — ใส่ให้ตรงเป๊ะกับชื่อในเกม (พิมพ์ใหญ่/เล็กก็ได้)
  5. กดปุ่ม Pay Now — ระบบจะพาไปหน้าชำระเงินของ Stripe
สำคัญ — ใส่ username ให้ถูก

Whitelist จะถูกผูกกับ username ที่กรอกในขั้นตอนนี้เท่านั้น ถ้ากรอกผิดเกมจะยังไม่อนุญาตให้เข้า — ต้องติดต่อแอดมินเพื่อแก้ไข

ปุ่มทดลองใช้งานฟรี (ถ้าเปิดให้ทดลอง)

บางสินค้ามีปุ่ม Try free — กดเพื่อทดลองใช้งานฟรีไม่กี่นาที (ไม่ต้องชำระเงิน):

  • ต้องกรอก username ก่อน
  • 1 username ทดลองได้ 1 ครั้งต่อ 24 ชั่วโมง
  • ถ้าทดลองสำเร็จจะขึ้นแบนเนอร์สีเขียวบอกเวลาหมดอายุ
  • ถ้า error จะขึ้นแบนเนอร์สีแดง พร้อมเหตุผล เช่น "ทดลองไปแล้วในช่วง 24 ชม. นี้"

1.4 ขั้นตอนการชำระเงินกับ Stripe

เมื่อกด Pay Now ระบบจะส่งคุณไปที่หน้าเว็บของ Stripe (ผู้ให้บริการรับชำระเงิน) — สังเกตว่า URL จะเปลี่ยนเป็น checkout.stripe.com

ถ้าเลือก PromptPay

  1. หน้า Stripe จะแสดง QR Code — เปิดแอปธนาคารบนมือถือแล้วสแกน
  2. ตรวจสอบยอดเงินและกดยืนยันในแอปธนาคาร
  3. เมื่อจ่ายเงินสำเร็จ Stripe จะพาคุณกลับมาที่หน้า Success ของ Judyshop อัตโนมัติ
PromptPay อาจใช้เวลายืนยันสักครู่

หลังจ่ายแล้วบางครั้ง Whitelist จะยังไม่เพิ่มทันที — รอสักครู่แล้วกด Refresh หน้า Success

ถ้าเลือกบัตรเครดิต/เดบิต

  1. กรอกหมายเลขบัตร · วันหมดอายุ · CVV
  2. กดปุ่ม Pay
  3. ธนาคารอาจให้ยืนยัน OTP — กรอกรหัสที่ได้รับทาง SMS
  4. เมื่อสำเร็จจะพากลับมาหน้า Success ทันที

ถ้ายกเลิกหรือปิดหน้า

ระหว่างอยู่หน้า Stripe คุณสามารถกดปุ่ม ← Back ของ Stripe หรือปิดแท็บได้ — จะถูกพาไปหน้า Cancel ของ Judyshop และไม่มีการเรียกเก็บเงิน

1.5 หน้าหลังการชำระเงิน

หน้า Success — ชำระเงินสำเร็จ

URL: /success?session_id=cs_xxx...

ส่วนบน — สรุปคำสั่งซื้อ

  • ไอคอนถูกใหญ่ (เครื่องหมายติ๊ก) สีชมพู
  • ข้อความ "ขอบคุณค่ะ!" หรือ "Thank you!"
  • ตารางสรุป: ชื่อสินค้า · แผน · Username · ยอดที่ชำระ · ใช้งานได้ถึง (วันที่ หรือ "ตลอดชีพ ∞")

ส่วนล่าง — ไฟล์ที่ดาวน์โหลดได้ (ถ้ามี)

  • Preset Files — ไฟล์ตั้งค่าสำหรับโปรแกรมต่างๆ — คลิกปุ่ม Download เพื่อเซฟ
  • Gift Overlays — รูปภาพ Gift Box หลายภาพ — คลิกปุ่ม Save แต่ละชิ้น หรือคลิกที่รูปเพื่อดูขนาดใหญ่
หากเปิดลิงก์ Success จากเบราว์เซอร์อื่น

ระบบรู้ว่าคุณไม่ใช่ผู้สั่งซื้อ — จะไม่แสดงรายละเอียดและไม่ให้ดาวน์โหลด เพื่อความปลอดภัย ให้กลับมาเปิดลิงก์จากเบราว์เซอร์เครื่องเดิมที่ใช้สั่งซื้อ

ถ้ายังไม่เห็นรายการสั่งซื้อ

  • Stripe อาจกำลังประมวลผลอยู่ — รอ 10–30 วินาทีแล้วกด Refresh
  • ถ้าเกิน 5 นาทียังไม่เห็น — ติดต่อแอดมินทาง Discord พร้อมแคป Session ID

หน้า Cancel — ยกเลิกการชำระเงิน

URL: /cancel

  • ไอคอน X สีเทา
  • ข้อความยืนยันว่าไม่มีการเรียกเก็บเงิน
  • ปุ่ม Try Again กลับไปหน้า Shop · ปุ่ม Home กลับหน้าแรก

1.6 ตรวจสอบ Whitelist (Check)

หน้าตรวจสอบสิทธิ์เข้าเกมที่ /check

วิธีใช้

  1. กรอก Roblox Username
  2. เลือก สินค้า ที่ต้องการเช็ค (หรือเลือก "Any product" เพื่อค้นทุกสินค้า)
  3. กด ค้นหา

ผลลัพธ์ที่เป็นไปได้

สถานะไอคอนความหมาย
ACTIVEเครื่องหมายติ๊กเขียวมีสิทธิ์เข้าเกม — แสดงวันหมดอายุและจำนวนวันที่เหลือ
EXPIREDนาฬิกาทรายเคยมีสิทธิ์แต่หมดอายุแล้ว — แสดงวันที่หมดอายุ
ไม่พบไอคอนแดงไม่มีข้อมูล username นี้ในระบบ
LIFETIME∞ ตลอดชีพใช้งานได้ตลอดชีพ
URL แชร์ได้

หลังค้นหา URL จะมีพารามิเตอร์ เช่น /check?username=xxx&productId=yyy — คัดลอกแชร์ให้คนอื่นเห็นผลเดียวกันได้

1.7 News

News — ข่าวสารและประกาศ

หน้าที่ /news รวมข่าวจากแอดมิน เช่น อัพเดทใหม่ · กิจกรรม · ปิดปรับปรุง

  • การ์ดข่าวเรียงจากใหม่ → เก่า
  • แต่ละข่าวมี: ป้ายหมวด (Update / Announce / Event / Maintenance), หัวข้อ, สรุปสั้น, วันที่
  • คลิกการ์ดเพื่ออ่านรายละเอียดเต็ม (ถ้ามี)

แต่ละหัวข้อคลิกเพื่อขยายดูคำตอบ (Accordion) — ปุ่ม "Join Discord" ที่ด้านล่างสำหรับติดต่อทีมงานโดยตรง

ส่วนที่ 2

คู่มือสำหรับผู้ดูแลระบบ (Admin)

ส่วนนี้สำหรับแอดมินที่ใช้หลังบ้านในการจัดการสินค้า · ออเดอร์ · Whitelist · ประกาศ · ข่าวสาร · ตั้งค่าระบบ และตรวจสอบ Audit Log

การเข้าถึง

เข้าได้เฉพาะ Admin เท่านั้น — เปิด /admin แล้วระบบจะพาไปหน้า Login อัตโนมัติ Session อยู่ได้ 8 ชั่วโมงก่อนต้อง Login ใหม่

2.1 Login และโครงสร้างเมนู Admin

หน้า Login

URL: /admin/login

  • กรอก อีเมล และ รหัสผ่าน
  • กดปุ่ม Sign in
  • เข้าได้สำเร็จจะพาไปหน้า Dashboard
  • ถ้าผิด — ขึ้นกล่องสีแดงพร้อมข้อความ

ถ้า Login อยู่แล้วแต่เปิดหน้านี้อีกครั้ง ระบบจะรีไดเรกต์ไป Dashboard อัตโนมัติ

โครงสร้างเมนู Admin

แถบบน (Top Bar)

  • โลโก้ "Judy Shop Admin" (ซ้าย) — คลิกกลับ Dashboard
  • อีเมลของแอดมินที่กำลัง login
  • ปุ่ม View site — เปิดหน้าร้านจริงในแท็บใหม่
  • ปุ่มเปลี่ยนธีม (สว่าง/มืด)
  • ปุ่ม Logout — ออกจากระบบ

แถบข้างซ้าย (Sidebar)

  • Overview: Dashboard
  • Catalogue: Products
  • Operations: Whitelist · Transactions · Announcements · News
  • System: Audit Log · Settings

บนมือถือ Sidebar จะซ่อนอยู่ในปุ่ม Hamburger ที่ Top Bar

2.2 Dashboard — ภาพรวม KPI

URL: /admin — หน้าหลักของ Admin แสดงตัวเลขสำคัญใน 30 วันล่าสุด เปรียบเทียบกับช่วง 30 วันก่อนหน้า

แถว KPI ด้านบน (4 การ์ด)

การ์ดแสดงอะไร
Revenueยอดขายรวม (THB) · % การเปลี่ยนแปลงเทียบช่วงก่อน
Paid Ordersจำนวนออเดอร์ที่จ่ายเงินแล้ว · เทียบช่วงก่อน
New Customersลูกค้าใหม่ · เทียบช่วงก่อน
Active Whitelistสิทธิ์เข้าเกมที่ยังใช้งานได้ · แสดงสัดส่วน Lifetime vs Timed

กราฟและตารางด้านล่าง

  • Revenue trend chart — กราฟเส้นแสดงยอดขายรายวัน 30 วัน
  • Top Products — กราฟแท่งแนวนอน 5–10 อันดับสินค้าขายดี (ยอดและจำนวนออเดอร์)
  • Whitelist Breakdown — กราฟวงกลม: Lifetime · Active · Expired
  • Order Status — กราฟวงกลม: Paid · Pending · Failed · Refunded
  • Recent Orders — 8 ออเดอร์ล่าสุด คลิก "View all →" ไปหน้า Transactions

2.3 Products — รายการสินค้า

  • ปุ่มมุมขวาบน + New product

ลบสินค้า

กดไอคอนถังขยะ → กล่องยืนยัน "Delete '[ชื่อสินค้า]'?" → กดยืนยัน → ระบบจะลบ:

  • ตัวสินค้า
  • Plans · Images · Gift Overlays · Presets ทั้งหมด
  • Whitelist ที่ผูกกับสินค้านั้น
ลบแล้วกู้คืนไม่ได้

การลบเป็น Cascade delete — ข้อมูลที่เชื่อมโยงหายหมด ถ้าต้องการแค่ซ่อนจากร้านให้ใช้ Toggle "Active" แทน

2.4 Products — สร้างใหม่และแก้ไข

สร้างสินค้าใหม่

URL: /admin/products/new — Form กรอกข้อมูลพื้นฐาน

ฟอร์มประกอบด้วย

Slug
คีย์ที่ใช้ใน URL เช่น judy-legend — ตัวเล็กและขีดกลางเท่านั้น (ถ้าเว้นว่างจะสร้างจากชื่ออัตโนมัติ)
Game ID
Roblox game ID หรือ URL เกม — ระบบดึงเลขออกมาเอง
Badge
เลือกได้: ไม่มี / HOT / NEW / SALE
Display order
เลข 0–99999 ยิ่งน้อยยิ่งแสดงก่อน
Active / Coming soon
สลับเปิด-ปิดและสถานะ Coming Soon
Name (EN) + Name (TH)
ชื่อเต็ม — ใช้แสดงในการ์ดสินค้า
Short Name (EN) + Short Name (TH)
ชื่อย่อ — สำหรับที่แคบ
Description (EN) + Description (TH)
คำบรรยายเต็ม — Rich text editor รองรับ Bold/Italic/Link
Short Description (EN) + Short Description (TH)
คำโปรยสั้น — ใช้ในการ์ดสินค้า

กดปุ่ม Create product → ระบบสร้างและพาไปหน้า Edit อัตโนมัติ

แก้ไขสินค้า (5 แท็บ)

URL: /admin/products/[id] — มี 5 แท็บให้สลับ:

แท็บ 1 — Details

เหมือนฟอร์มสร้างใหม่ + ตัวเลือกพิเศษ:

  • Trial Enabled — เปิด/ปิดให้ลูกค้ากดทดลองฟรี
  • Trial Minutes — ระยะเวลาทดลอง เช่น 10 นาที

กด Save — ขึ้น Toast "Product updated ✓"

แท็บ 2 — Images

  • อัพโหลด: drag-and-drop หรือคลิกเลือกไฟล์ — ใส่ Alt text (EN/TH) ได้
  • รายการรูป: ตัวเลขลำดับซ้าย + Thumbnail + URL + ปุ่มขึ้น/ลงเพื่อจัดลำดับ
  • Set as thumbnail — กำหนดให้รูปนี้เป็นรูปหลัก (ปุ่มดาวฟ้า)
  • Delete — ลบรูป (มี Confirm dialog)

รูปแรกในรายการคือรูปที่ลูกค้าเห็นในการ์ดสินค้าที่หน้า Shop

แท็บ 3 — Plans

กำหนดราคาและระยะเวลาให้กับสินค้า — ลูกค้าจะเลือกแผนตรงนี้ตอนซื้อ

Label (EN) + Label (TH)
ชื่อแผน เช่น "30 Days", "Lifetime", "ตลอดชีพ"
Duration (days)
ระยะเวลาเป็นวัน — ติ๊ก Lifetime เพื่อใช้งานตลอดชีพแทน
Price (THB) + Price (USD)
ราคาบาทและดอลลาร์ — ติ๊ก Auto USD ให้ระบบคำนวณ USD จาก THB อัตโนมัติ
Active toggle
ปิดเพื่อซ่อนแผนนี้ชั่วคราว (ไม่ต้องลบ)
ต้องมีอย่างน้อย 1 plan

สินค้าที่ไม่มี active plan เลย จะไม่แสดงในร้านโดยอัตโนมัติ (ขายไม่ได้)

แท็บ 4 — Overlays (Gift Boxes)

รูปประดับ Gift Box สำหรับใช้ในเกม:

  • อัพโหลดรูป PNG/JPG พร้อมตั้งชื่อ Gift (EN+TH)
  • เรียงลำดับด้วยปุ่มขึ้น/ลง
  • Active toggle เปิด/ปิดแสดง
  • ลบได้พร้อม Confirm dialog

แท็บ 5 — Presets

ไฟล์ตั้งค่าให้ลูกค้าดาวน์โหลด (เช่น config สำหรับ OBS):

อัพโหลดไฟล์
Drag-and-drop หรือคลิกเลือก — ระบบเก็บชื่อไฟล์และขนาดอัตโนมัติ
Name (EN)
ชื่อ Preset ที่ลูกค้าเห็น
Target Program
ระบุโปรแกรมที่ใช้ เช่น "OBS", "Streamlabs" (ไม่บังคับ)
Active toggle
ปิดเพื่อหยุดให้ดาวน์โหลด

ปุ่ม Open แต่ละแถวเปิดไฟล์ในแท็บใหม่เพื่อตรวจสอบ

2.5 Transactions — ออเดอร์และการคืนเงิน

URL: /admin/transactions — รายการออเดอร์ทั้งหมดและการคืนเงิน

ตัวกรอง (Filter)

แถวปุ่ม Pill ด้านบน: All PAID PENDING FAILED REFUNDED — คลิกเพื่อกรอง

คอลัมน์ในตาราง

คอลัมน์ความหมาย
StatusBadge สี: PAID (เขียว) / PENDING (ส้ม) / FAILED (เทา) / REFUNDED (ชมพู)
Product · Planชื่อสินค้า (ตัวหนา) + ชื่อแผน
UserRoblox username (ฟอนต์ Mono)
MethodCard หรือ PromptPay
Amountยอด ฿ พร้อมเครื่องหมายสกุลเงิน
StripePayment Intent ID — hover ดูเต็ม
Dateวันที่และเวลา
Actionsปุ่ม Refund (เฉพาะออเดอร์ PAID ที่มี Stripe ID)

50 แถวต่อหน้า · มี Pagination ด้านล่าง

ขั้นตอนการคืนเงิน (Refund)

  1. คลิกปุ่ม Refund ที่แถวออเดอร์
  2. กล่องยืนยันจะเปิดขึ้น พร้อมรายละเอียดออเดอร์:
    • Product · Plan · Username · Amount · Payment ID · Order ID
    • กล่องสีชมพูบอกผลลัพธ์: เงินคืนเข้า Card/PromptPay (1–10 วันทำการ) · Whitelist ถูกตัดทันที · Order เปลี่ยนเป็น REFUNDED + Audit log
  3. กรอก เหตุผล (ไม่บังคับ แต่แนะนำ — สูงสุด 500 ตัวอักษร) เช่น "ลูกค้าจ่ายผิด username", "ขอคืนเอง"
  4. กด Confirm refund — หรือ Cancel/กด ESC เพื่อยกเลิก
  5. เมื่อสำเร็จ Toast เขียวขึ้น: "Refund issued · Stripe refunded · whitelist revoked · order marked REFUNDED"
การคืนเงินทำกลับไม่ได้

เมื่อยืนยันแล้วระบบจะคืนเงินจริงผ่าน Stripe และตัด Whitelist ทันที — ตรวจสอบให้แน่ใจก่อนกด Confirm

ข้อผิดพลาดที่อาจเจอ และวิธีแก้

Errorวิธีแก้
DB unreachableฐานข้อมูล (Supabase) ปิดอยู่ — เปิด Supabase Dashboard ให้กลับมาทำงาน
No payment intentOrder ไม่มี payment_intent — คืนเงินผ่าน Stripe Dashboard ตรงๆ แล้ว mark REFUNDED ในระบบ
Stripe rejectedStripe ปฏิเสธ — ตรวจ Payment Intent ใน Stripe Dashboard
Order not paidคืนเฉพาะออเดอร์ที่ PAID เท่านั้น
Already refundedคืนไปแล้วก่อนหน้า

2.6 Whitelist — จัดการสิทธิ์เข้าเกม

URL: /admin/whitelist — เพิ่ม/ลบ/ต่ออายุ Whitelist แบบ Manual

ฟอร์มเพิ่ม Whitelist (แผงบน)

Product
เลือกสินค้า (Dropdown) — บังคับ
Roblox Username
username แบบเป๊ะ (ตัวพิมพ์ใหญ่/เล็กไม่แมตเตอร์) สูงสุด 100 ตัว — บังคับ
Duration (days)
จำนวนวันที่ใช้งานได้ 1–36,500 — ปิดถ้าติ๊ก Lifetime
Label
โน้ตภายในที่เฉพาะแอดมินเห็น เช่น "promo", "refund", "VIP" (สูงสุด 200 ตัว ไม่บังคับ)
Lifetime toggle
เปิดเพื่อให้ใช้งานตลอดชีพ (ไม่มีวันหมด)

ค้นหาและกรอง

  • ช่อง Search — ค้นหา username (ไม่สนใจตัวพิมพ์ใหญ่/เล็ก)
  • Dropdown Product — กรองตามสินค้า
  • ปุ่ม Apply และ Reset

คอลัมน์ในตาราง

คอลัมน์ความหมาย
User · ProductUsername (Mono) + ชื่อสินค้า
StatusLifetime / Active / Expired
SourceSTRIPE (จากการซื้อ) หรือ MANUAL (Admin เพิ่ม)
Addedวันที่เพิ่ม
Expiresวันที่หมดอายุ + จำนวนวันที่เหลือ (สีเขียวถ้าเหลือ >7 วัน, สีส้มถ้าเหลือ 1–7 วัน, สีแดงถ้าหมดแล้ว) หรือ "Lifetime ∞"
Labelโน้ตภายในหรือ "—"
Actionsปุ่มลบ (X สีแดง)
การลบ Whitelist ตัดสิทธิ์ทันที

กดลบแล้วลูกค้าจะเข้าเกมไม่ได้ภายในไม่กี่วินาที — ใช้ระมัดระวัง สำหรับเคสคืนเงินใช้ปุ่ม Refund ที่หน้า Transactions จะเหมาะกว่า (จะคืนเงินจริงด้วย)

2.9 Settings — ตั้งค่าระบบ

URL: /admin/settings — ตั้งค่าระดับเว็บไซต์

Credit Card Surcharge (%)

  • กรอกตัวเลข 0–100 (ทศนิยม 2 หลัก) เช่น 2.5
  • ใช้กับการชำระด้วยบัตรเครดิตเท่านั้น — PromptPay ไม่เก็บ
  • มี Live Preview ด้านล่าง: ใช้ตะกร้าตัวอย่าง ฿1,000 แสดงยอดที่ลูกค้าจะจ่ายจริง

กด Save — ใช้ผลทันทีกับ checkout ครั้งถัดไป

2.10 Audit Log — บันทึกการใช้งาน

URL: /admin/audit — บันทึกทุกการกระทำของ Admin (ไม่สามารถลบหรือแก้)

ค้นหาและกรอง

  • Search — ค้นหา email, action name, target ID
  • Action Filter — Dropdown action ทั้งหมด เช่น refund_order, delete_announcement
  • ปุ่ม Filter และ Reset

คอลัมน์

คอลัมน์ความหมาย
Whenวันเวลาแบบเต็มมีวินาที
Adminอีเมลของแอดมินที่ทำ (หรือ "system" ถ้าระบบทำเอง)
Actionชื่อ action เช่น refund_order, toggle_announcement_active
Targetเป้าหมายรูปแบบ TYPE:ID เช่น order:abc123
Payloadรายละเอียดเป็น JSON เช่น เหตุผลคืนเงิน, ค่าก่อน/หลังแก้ไข
IPIP ของแอดมินตอนทำ

ตัวอย่างที่บันทึก

  • คืนเงิน → refund_order + payload เก็บเหตุผล, ยอด, Stripe ID
  • ลบ Announcement → delete_announcement + target ID
  • เพิ่ม Whitelist → create_whitelist + username, ระยะเวลา, label
  • แก้ Settings → update_settings + ค่าก่อน/หลัง
Audit Log ไม่สามารถแก้ไขหรือลบได้

บันทึกถาวรสำหรับ accountability และ Compliance — ทุกการกระทำของแอดมินถูกบันทึกอัตโนมัติ

A. Troubleshooting

ฝั่งลูกค้า

ปัญหาวิธีแก้
จ่ายเงินแล้วแต่ Whitelist ไม่เพิ่มรอ 1–5 นาทีแล้ว Refresh หน้า Success ถ้ายังไม่ได้ติดต่อ Discord พร้อม Session ID
กรอก username ผิดติดต่อแอดมินใน Discord — ระบบไม่ให้แก้เอง
ใช้ทดลองฟรีไม่ได้ใช้ไปแล้วใน 24 ชม. ที่ผ่านมา หรือสินค้าไม่เปิด trial — รอ 24 ชม. แล้วลองใหม่
เกมไม่ให้เข้าทั้งที่มี Whitelistตรวจ username ในเกมว่าตรงกับที่ซื้อ · เช็ควันหมดอายุที่ /check
เปิดลิงก์ Success แล้วไม่เห็นรายละเอียดเปิดผิดเบราว์เซอร์ — ต้องเปิดจากเครื่องเดียวกับที่สั่งซื้อ

ฝั่ง Admin

ปัญหาวิธีแก้
Refund ขึ้น "DB unreachable"Supabase อาจถูกพัก (free tier) — เปิด Supabase Dashboard ให้กลับมาทำงาน
Refund ขึ้น "No payment intent"Order ไม่มี payment_intent — ทำใน Stripe Dashboard แล้วค่อยมา mark REFUNDED
สินค้าไม่ขึ้นในร้านตรวจ Active toggle · ตรวจว่ามี active plan อย่างน้อย 1 อัน
Image upload ไม่ขึ้นไฟล์ต้องไม่เกิน 6 MB และเป็น JPG/PNG/WebP
Announcement ตั้งแล้วไม่ขึ้นตรวจ Start Date / End Date / Active toggle
