คู่มือสำหรับ
ผู้ใช้ทั่วไป

เนื้อหาในส่วนนี้สำหรับลูกค้าที่ต้องการเรียนรู้วิธีซื้อสิทธิ์ Whitelist ขั้นตอนการชำระเงิน การรับสิทธิ์เข้าเกม และวิธีติดตามข่าวสารจากร้านค้า ใช้งานได้ทุกอุปกรณ์

บทที่ 1.1

ภาพรวมและเมนูหลัก

แถบเมนูด้านบน (Navbar)

ติดอยู่ที่ด้านบนของทุกหน้า เลื่อนตามขึ้น/ลงไม่หาย ประกอบด้วย:

  • โลโก้ Judy Shop (ซ้ายสุด) — คลิกเพื่อกลับหน้าแรก
  • เมนูหลัก: Home · Shop · News — เมนูที่กำลังเปิดอยู่จะมีจุดสีชมพูใต้ชื่อ
  • ปุ่มสลับภาษา TH / EN — ปุ่มสีชมพูคือภาษาที่เลือกอยู่ (URL จะเปลี่ยนเป็น /en/...)
  • ปุ่มเปลี่ยนธีม สว่าง / มืด
  • ไอคอน Discord และโซเชียลอื่น — เปิดในแท็บใหม่
บนมือถือ

เมนู Home / Shop / News จะซ่อนอยู่ในปุ่ม Hamburger (สามขีด) ทางขวา แตะเพื่อเปิด Drawer

Footer ด้านล่าง

เลื่อนสุดหน้าจะเจอ Footer ประกอบด้วย: ลิขสิทธิ์ © ปีปัจจุบัน, ลิงก์เมนูเสริม, ไอคอนโซเชียล

บทที่ 1.2

หน้าแรกและป็อปอัพประกาศ

URL: / หรือ /en

สิ่งที่จะเห็นในหน้าแรก

  • โลโก้ JUDY SHOP ขนาดใหญ่ มีเอฟเฟกต์เรืองแสง (ชมพู + ฟ้า)
  • คำโปรย ใต้โลโก้
  • ปุ่ม Shop Now — ไปที่หน้า Shop
  • ปุ่ม Join Discord — เปิด Discord ในแท็บใหม่
  • Mascot น้อง Judy ลอยเคลื่อนไหวเบาๆ ที่ด้านล่าง

ป็อปอัพประกาศ (ถ้ามี)

  • 3 รูปแบบ: รูปภาพอย่างเดียว · ข้อความอย่างเดียว · ทั้งรูปและข้อความ
  • ปิดได้ 2 วิธี: กดปุ่ม X มุมขวาบน หรือคลิกพื้นที่มืดด้านหลัง
  • แสดงทุกครั้งที่เข้าหน้านี้ (ไม่จำว่าปิดไปแล้ว) เพื่อไม่พลาดประกาศสำคัญ
บทที่ 1.3

Shop — เลือกสินค้าและสั่งซื้อ

URL: /shop

หน้า Shop หลัก

  • หัวเรื่อง SHOP พร้อมคำบรรยายใต้หัวเรื่อง
  • กริดสินค้า: มือถือ 2 คอลัมน์ · แท็บเล็ต 3 · คอมพิวเตอร์ 4

การ์ดสินค้าแต่ละใบ

  • รูปสินค้า (สัดส่วน 5:3) เคลื่อนซูมเล็กน้อยเมื่อ hover
  • ป้ายแบดจ์มุมขวาบน HOT NEW SALE (ถ้ามี)
  • หากยังไม่เปิดขาย จะมีแถบทาบ Coming Soon
  • ชื่อสินค้า + คำบรรยายสั้น (2 บรรทัด)
  • ราคาเริ่มต้น From ฿XXX / $XX (ราคาแผนถูกสุด)
  • หากสินค้ายังไม่มีรูป จะแสดงไอคอน "ไม่มีรูป" แทน

เมื่อคลิกสินค้า — Product Modal

  1. ดูรูปสินค้า เลื่อนซ้าย/ขวาในแกลเลอรี
  2. เลือกแพ็กเกจ เช่น "30 วัน", "60 วัน", "ตลอดชีพ"
  3. เลือกวิธีชำระเงิน: PromptPay (ไม่มีค่าธรรมเนียม) หรือ บัตรเครดิต (มีค่าธรรมเนียมเพิ่ม)
  4. กรอก Roblox Username ให้ตรงเป๊ะกับชื่อในเกม
  5. กดปุ่ม Pay Now ระบบจะพาไปหน้าชำระเงินของ Stripe
สำคัญ — ใส่ Username ให้ถูก

Whitelist จะถูกผูกกับ Username ที่กรอกในขั้นตอนนี้เท่านั้น ถ้ากรอกผิดเกมจะยังไม่อนุญาตให้เข้า ต้องติดต่อแอดมินเพื่อแก้ไข

ปุ่มทดลองใช้งานฟรี

บางสินค้ามีปุ่ม Try free ทดลองใช้ฟรีไม่กี่นาที:

  • ต้องกรอก Username ก่อน
  • 1 username ทดลองได้ 1 ครั้งต่อ 24 ชั่วโมง
  • หากสำเร็จขึ้นแบนเนอร์เขียวบอกเวลาหมดอายุ
บทที่ 1.4

การชำระเงินผ่าน Stripe

เมื่อกด Pay Now ระบบจะส่งไปที่หน้าเว็บของ Stripe สังเกตว่า URL จะเปลี่ยนเป็น checkout.stripe.com

กรณีเลือก PromptPay

  1. หน้า Stripe จะแสดง QR Code เปิดแอปธนาคารแล้วสแกน
  2. ตรวจสอบยอดและกดยืนยันในแอปธนาคาร
  3. เมื่อชำระสำเร็จ Stripe จะพากลับมาที่หน้า Success อัตโนมัติ
PromptPay อาจใช้เวลายืนยันสักครู่

หลังชำระแล้วบางครั้ง Whitelist จะยังไม่เพิ่มทันที กรุณารอสักครู่แล้วรีเฟรชหน้า Success อีกครั้ง

กรณีเลือกบัตรเครดิต / เดบิต

  1. กรอกหมายเลขบัตร · วันหมดอายุ · CVV
  2. กดปุ่ม Pay
  3. ธนาคารอาจให้ยืนยัน OTP กรอกรหัสที่ได้รับทาง SMS
  4. เมื่อสำเร็จจะพากลับมาหน้า Success ทันที

กรณียกเลิกหรือปิดหน้า

กดปุ่ม ← Back หรือปิดแท็บได้ระหว่างอยู่หน้า Stripe จะไปหน้า Cancel และไม่มีการเรียกเก็บเงิน

บทที่ 1.5

หน้า Success / Cancel

หน้า Success — ชำระเงินสำเร็จ

URL: /success?session_id=cs_xxx...

ส่วนบน — สรุปคำสั่งซื้อ

  • ไอคอนเครื่องหมายติ๊กสีชมพู
  • ข้อความ "ขอบคุณสำหรับการสั่งซื้อ"
  • ตารางสรุป: ชื่อสินค้า · แผน · Username · ยอดที่ชำระ · ใช้งานได้ถึง

ส่วนล่าง — ไฟล์ดาวน์โหลด (ถ้ามี)

  • Preset Files ไฟล์ตั้งค่าสำหรับโปรแกรมต่างๆ
  • Gift Overlays รูปภาพ Gift Box
ป้องกัน Link หลุด

ถ้าเปิดลิงก์ Success จากเบราว์เซอร์อื่น (ไม่ใช่เครื่องที่สั่งซื้อ) ระบบรู้ทันที จะไม่แสดงรายละเอียดและไม่ให้ดาวน์โหลด เพื่อความปลอดภัย

หน้า Cancel — ยกเลิกการชำระเงิน

URL: /cancel — ไอคอน X เทา + ข้อความยืนยันว่าไม่มีการเรียกเก็บเงิน + ปุ่ม Try Again / Home

บทที่ 2.1

Login และโครงสร้างเมนู Admin

หน้า Login

URL: /admin/login — กรอก email + password → กดปุ่ม Sign in

วิธีสร้าง Admin Account

รันคำสั่งบน server:

npm run admin:create -- <email> <password> ["ชื่อ"]

ตัวอย่าง:

npm run admin:create -- admin@judygamestudio.com "P@ssw0rd!" "Admin"
  • Password ขั้นต่ำ 8 ตัวอักษร
  • หาก email มีอยู่แล้ว ระบบจะอัพเดทรหัสผ่านให้ (ใช้รีเซ็ตรหัสได้)
ระวัง CMD ไม่รู้จัก single quote

บน Windows CMD ใช้ double quote หรือไม่ใส่ quote — single quote '...' จะถูกตีความเป็น literal character ทำให้ password ที่ hash จริงต่างจากที่กรอกตอน login

  • ARR 3.0: iis.net/downloads/microsoft/application-request-routing

สร้าง site:

  • Sites → Add Website
  • Physical path: C:\inetpub\wwwroot\judyshop
  • Binding: http port 80, hostname = yourdomain.com

ใส่ web.config:

<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="ReverseProxyToNode" stopProcessing="true">
          <match url="(.*)" />
          <action type="Rewrite" url="http://localhost:3000/{R:1}" />
          <serverVariables>
            <set name="HTTP_X_FORWARDED_PROTO" value="https" />
            <set name="HTTP_X_FORWARDED_HOST" value="{HTTP_HOST}" />
            <set name="HTTP_X_FORWARDED_FOR" value="{REMOTE_ADDR}" />
          </serverVariables>
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>

อนุญาต serverVariables (PowerShell Admin):

& "$env:windir\system32\inetsrv\appcmd.exe" set config -section:system.webServer/rewrite/allowedServerVariables /+"[name='HTTP_X_FORWARDED_PROTO']" /commit:apphost
& "$env:windir\system32\inetsrv\appcmd.exe" set config -section:system.webServer/rewrite/allowedServerVariables /+"[name='HTTP_X_FORWARDED_HOST']" /commit:apphost

บทที่ 4.2

Cloudflare + Origin Certificate

1) ตั้ง DNS

  • A record yourdomain.com → IP VPS — 🟠 Proxied
  • A record www.yourdomain.com → IP เดียวกัน — 🟠 Proxied

2) สร้าง Origin Certificate

SSL/TLS → Origin Server → Create Certificate:

  • RSA (2048)
  • Hostnames: yourdomain.com, *.yourdomain.com
  • Validity: 15 years

Copy เก็บ Origin Certificate + Private key (key แสดงครั้งเดียว)

3) แปลงเป็น PFX + Import

บันทึก cert + key เป็นไฟล์บน VPS:

C:\ssl\origin.crt
C:\ssl\origin.key

แปลงเป็น PFX:

& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs12 -export `
  -out C:\ssl\origin.pfx `
  -inkey C:\ssl\origin.key `
  -in C:\ssl\origin.crt `
  -name "Cloudflare Origin"

Import เข้า Cert Store (Run as Admin):

$pwd = Read-Host -AsSecureString "Enter PFX password"
Import-PfxCertificate -FilePath "C:\ssl\origin.pfx" `
  -CertStoreLocation "Cert:\LocalMachine\My" -Password $pwd

4) Bind cert ที่ IIS port 443

$cert = Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.FriendlyName -eq "Cloudflare Origin" } | Select-Object -First 1
Import-Module WebAdministration
New-WebBinding -Name "judyshop" -Protocol "https" -Port 443 -HostHeader "yourdomain.com" -SslFlags 1
$binding = Get-WebBinding -Name "judyshop" -Protocol "https"
$binding.AddSslCertificate($cert.Thumbprint, "My")

เปิด firewall port 443:

New-NetFirewallRule -DisplayName "HTTPS" -Direction Inbound -Protocol TCP -LocalPort 443 -Action Allow

5) ตั้ง Cloudflare SSL mode

SSL/TLS → Overview → Full (strict)

Edge Certificates → เปิด:

  • ✅ Always Use HTTPS
  • ✅ Automatic HTTPS Rewrites
การ Remote เข้า VPS หลังเปิด Proxied

เมื่อ DNS เป็น Proxied (🟠) — port 3389 (RDP) ใช้ yourdomain.com ไม่ได้แล้ว ให้ RDP ด้วย IP จริงของ VPS หรือสร้าง subdomain server.yourdomain.com เป็น DNS only (ไม่ Proxied) เพื่อ RDP ได้

ผู้ให้บริการ Payment Gateway รับชำระทั้งบัตรเครดิตและ PromptPay
Session ID
รหัสคำสั่งซื้อจาก Stripe (เริ่มด้วย cs_)
Payment Intent
รหัสการชำระเงินจาก Stripe (เริ่มด้วย pi_) ใช้สำหรับ Refund
Audit Log
บันทึกถาวรของการกระทำของแอดมิน ไม่สามารถลบหรือแก้ได้
Trial
การทดลองใช้งานสินค้าฟรีในระยะเวลาสั้นๆ ก่อนซื้อจริง
Whitelist Source
ที่มา: STRIPE (จากการซื้อ), MANUAL (Admin), PROMO, TRIAL, REFUND_REVERT
PM2
Process Manager สำหรับ Node.js รัน app เป็น background + auto restart
IIS
Internet Information Services — web server ของ Windows ใช้เป็น reverse proxy
Reverse Proxy
Server ที่รับ request จาก client แล้วส่งต่อให้ app ที่รันบน port ภายใน
Origin Certificate
SSL Certificate ของ Cloudflare ใช้คุยระหว่าง Cloudflare ↔ VPS เท่านั้น
ISR
Incremental Static Regeneration — Cache หน้าเว็บแล้ว regenerate เป็นช่วงๆ
Webhook
HTTP Endpoint ที่ผู้ให้บริการเรียกมาเพื่อแจ้งเหตุการณ์ (เช่น "ลูกค้าจ่ายเงินสำเร็จ")
